# LeetCode 48: Rotate Image (旋转图像)

## 1. 解题思路

这道题的核心要求是 **“原地”** 顺时针旋转 90 度，这意味着我们不能使用额外的矩阵来辅助，空间复杂度必须是 O(1)。围绕这个限制，有两种主流的经典解法。

### 思路一: 两次翻转法 (转置 + 水平翻转)

这种方法非常巧妙，将一个复杂的旋转操作分解为两次相对简单的翻转操作。这通常是更容易想到和实现的方法。

1.  **矩阵转置 (Transpose)**: 沿着主对角线（从左上到右下）进行翻转。简单来说，就是交换 `matrix[i][j]` 和 `matrix[j][i]`。
2.  **水平翻转 (Reflect)**: 对转置后的矩阵，以垂直中心线为轴，将每一行的元素进行左右翻转。简单来说，就是交换 `matrix[i][j]` 和 `matrix[i][n-1-j]`。

这两步操作结合起来，效果等同于顺时针旋转 90 度。

**循环边界要点**:
- **转置**: `for (int i = 0; i < n; i++) { for (int j = i; j < n; j++) { ... } }`。内层循环 `j` 从 `i` 开始，是为了避免重复交换，否则 `(i,j)` 和 `(j,i)` 会被交换两次，等于没变。
- **翻转**: `for (int i = 0; i < n; i++) { for (int j = 0; j < n / 2; j++) { ... } }`。内层循环 `j` 只需要到 `n/2`，因为是成对交换。

- **时间复杂度**: O(n²)，需要遍历矩阵两次。
- **空间复杂度**: O(1)，原地修改。

### 思路二: 分层旋转法 (原地四方交换)

这种方法更加直接，模拟了每个元素旋转后的最终位置，但对坐标计算和循环边界的要求更高。这也是你尝试的思路。

我们可以把矩阵想象成一个洋葱，由多个“口”字形的层嵌套而成。我们一次只处理一层，从最外层开始，直到最内层。

对于每一层，我们遍历其中一条边（比如上边），对边上的每个元素，找到它在旋转 90 度后对应的另外 3 个位置的元素，进行一次 **四方交换**。

**循环边界和坐标计算要点 (这是本题的难点与精髓)**:

- **外层循环 (分层)**: `for (int i = 0; i < n / 2; i++) { ... }`
  - `i` 代表当前处理的层数（0是最外层）。
  - 为什么到 `n / 2`？因为层是对称的，处理完 `n/2` 层，整个矩阵就都被覆盖了。

- **内层循环 (遍历层内元素)**: `for (int j = i; j < n - 1 - i; j++) { ... }`
  - `j` 代表当前层一条边上的元素索引。
  - 循环从 `i` 开始，到 `n - 1 - i` 结束（**不包含**）。`i` 是当前层的左边界，`n - 1 - i` 是右边界。
  - **为什么不包含右边界 `n-1-i`？** 因为一个角上的元素（比如左上角）会在一次四方交换中，带动其他三个角一起旋转。如果遍历到右边界，会导致同一个元素被重复计算，扰乱旋转结果。我们只需要遍历一条边上（不含一个端点）的点，就能带动整层所有点完成旋转。

- **四方交换的坐标**: 对于第 `i` 层的第 `j` 个元素，它的坐标是 `(i, j)`。它参与交换的四个位置是：
  - 左上: `(i, j)`
  - 右上: `(j, n - 1 - i)`
  - 右下: `(n - 1 - i, n - 1 - j)`
  - 左下: `(n - 1 - j, i)`
  
  通过一个 `temp` 变量即可完成这四个位置上值的轮换。

- **时间复杂度**: O(n²)，每个元素都作为四方交换的一员被访问一次。
- **空间复杂度**: O(1)，原地修改。

## 2. 思路来源 (如何想到)

1.  **"原地" (In-place) 约束**: 这是最强的信号。它直接否定了“创建一个新矩阵，然后根据 `new_matrix[j][n-1-i] = old_matrix[i][j]` 规则填充”的暴力解法。我们的思路必须转向如何在原矩阵上进行修改。
2.  **寻找规律**:
    - **方法一（数学/几何思路）**: 顺时针旋转90度这个操作，在几何上可以分解为什么？可以分解为“沿 y=x 轴翻转”+“沿 y 轴翻转”，这分别对应了程序中的“矩阵转置”和“水平翻转”。这需要一点几何/线性代数的直觉。
    - **方法二（直接模拟思路）**: 一个元素 `(i, j)` 旋转后会去哪里？通过画图可以发现 `(i, j) -> (j, n-1-i)`。那 `(j, n-1-i)` 又会去哪里？一路跟下去，会发现它形成了一个 4 个元素的交换闭环。这就自然地导向了“分层”和“四方交换”的思路。

这类问题都考察对矩阵坐标的精确控制和边界处理能力。

1.  **[LeetCode 54. Spiral Matrix](https://leetcode.cn/problems/spiral-matrix/)**: 螺旋矩阵 I。这道题是按螺旋顺序“读取”矩阵元素。它和本题的“分层”思想异曲同工，都需要维护 `top, bottom, left, right` 四个边界，并在一轮轮循环中向内收缩。是练习矩阵边界处理的绝佳题目。
2.  **[LeetCode 59. Spiral Matrix II](https://leetcode.cn/problems/spiral-matrix-ii/)**: 螺旋矩阵 II。这道题是按螺旋顺序“写入”一个矩阵。遍历和边界控制的逻辑与螺旋矩阵 I 完全一样。

## 4. 涉及的关键方法/API (Java)

本题是纯粹的算法逻辑题，几乎不涉及任何复杂的 API。

- **`int temp = ...`**: 交换变量时必需的临时存储。
- **`matrix[i][j]`**: 二维数组的索引访问。

核心能力是对 **嵌套 for 循环**、**循环边界** 和 **数组下标运算** 的熟练掌握。
