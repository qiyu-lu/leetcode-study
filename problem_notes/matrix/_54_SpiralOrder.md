# LeetCode 54: Spiral Matrix (螺旋矩阵)

## 1. 解题思路

这道题最核心且最直观的解法是 **模拟法 (Simulation)**，也常被称为 **“逐层剥离” (Layer-by-Layer)**。我们可以想象矩阵是由多个“回”字形嵌套组成的，我们的任务就是从最外层开始，顺时针地一层一层“剥”下所有元素。

具体实现上，我们通过维护四个变量来定义当前层的边界：
- `top`: 未遍历区域的上边界行号
- `bottom`: 未遍历区域的下边界行号
- `left`: 未遍历区域的左边界列号
- `right`: 未遍历区域的右边界列号

算法的主体是一个 `while` 循环，循环条件是 `top <= bottom && left <= right`，确保还有未遍历的区域存在。在循环内部，我们按顺序执行四步，模拟顺时针遍历一圈的过程：

1.  **从左到右 (遍历上边界)**: `for` 循环从 `left` 到 `right`，将 `matrix[top][i]` 依次加入结果列表。完成後 `top++`，上边界向下收缩。
2.  **从上到下 (遍历右边界)**: `for` 循环从 `top` 到 `bottom`，将 `matrix[i][right]` 依次加入结果列表。完成后 `right--`，右边界向左收缩。
3.  **从右到左 (遍历下边界)**: `for` 循环从 `right` 到 `left`，将 `matrix[bottom][i]` 依次加入结果列表。完成后 `bottom--`，下边界向上收缩。
4.  **从下到上 (遍历左边界)**: `for` 循环从 `bottom` 到 `top`，将 `matrix[i][left]` 依次加入结果列表。完成后 `left++`，左边界向右收缩。

**特别注意**: 对于非 n*n 的矩阵，当“层”被压缩成单行或单列时，上述的第 3 和第 4 步可能会遍历到已经走过的元素。因此，在第 2 步和第 3 步执行后，需要立即检查边界条件 (`if (top > bottom || left > right) break;`)，防止重复遍历。

### 复杂度分析
- **时间复杂度**: O(m * n)。其中 m 是矩阵的行数，n 是列数。每个元素都只会被访问一次。
- **空间复杂度**: O(1)。我们只使用了常数个变量作为边界指针。注意，返回的结果列表不计入额外空间复杂度。

## 2. 思路来源 (如何想到)

题目的描述“请按照 **顺时针螺旋顺序** ，返回矩阵中的所有元素”是本题最关键的引导。这种明确的、非标准的遍历顺序，直接排除了简单的双层 for 循环。

1.  **直观模拟**: “螺旋”这个词本身就暗示了一个路径。最自然的想法就是去模拟这个路径的行走过程。
2.  **路径分解**: 螺旋路径可以被分解为一系列的直线运动：向右 -> 向下 -> 向左 -> 向上 -> ...
3.  **边界控制**: 在每次走完一条边后，我们实际上是完成了一部分遍历，并且活动范围（未遍历的区域）缩小了。这就很自然地引出了使用边界变量 (`top`, `bottom`, `left`, `right`) 来定义当前活动范围，并在每走完一条边后收缩相应边界的想法。

因此，题目的引导非常直接，考察的是将一个具象化的路径转化为代码逻辑的能力。

## 3. 举一反三

1.  **[LeetCode 59. Spiral Matrix II](https://leetcode.cn/problems/spiral-matrix-ii/)**: 这是本题的逆向问题。它要求你生成一个 n*n 的矩阵，并用 1 到 n^2 的数字按螺旋顺序填充。解题的遍历逻辑和边界控制方法与本题完全相同，只是操作从“读”变成了“写”。
2.  **[LeetCode 48. Rotate Image](https://leetcode.cn/problems/rotate-image/)**: 旋转图像。虽然不是螺旋遍历，但高效的解法之一（原地旋转）也涉及到对矩阵进行分层处理，从外到内，每次处理一个“回”字形的四个角和四条边上的元素。这种“分层”的思想是共通的。
3.  **[LeetCode 2326. Spiral Matrix IV](https://leetcode.cn/problems/spiral-matrix-iv/)**: 本题的进阶版。要求将一个链表中的节点按螺旋顺序填充到矩阵中。遍历矩阵的逻辑不变，但数据源从矩阵本身变成了链表节点，需要同时维护矩阵指针和链表指针。

## 4. 涉及的关键方法/API (Java)

本题的算法实现本身不依赖复杂的库函数，主要考察的是逻辑控制能力。

- `java.util.List`: 用于定义返回结果的类型。
- `java.util.ArrayList`:
  - `new ArrayList<>()`: 创建一个列表实例来存储结果。
  - `list.add(element)`: 在遍历过程中，将矩阵中的元素添加到结果列表中。
- 循环控制: `while` 和 `for` 循环是实现此逻辑的核心。
