# 239. 滑动窗口最大值

## 题目链接

[https://leetcode.cn/problems/sliding-window-maximum/](https://leetcode.cn/problems/sliding-window-maximum/)

## 思路解析

这道题要求我们找到一个大小为 `k` 的滑动窗口在数组 `nums` 上从左到右移动时，每个窗口内的最大值。

#### 1. 暴力解法

最直观的想法是，遍历数组中的每一个可能的窗口，然后为每个窗口再遍历一次其中的 `k` 个元素，找到最大值。

- 第一个窗口从 `nums[0]` 到 `nums[k-1]`，找到最大值。
- 第二个窗口从 `nums[1]` 到 `nums[k]`，找到最大值。
- ...
- 最后一个窗口从 `nums[n-k]` 到 `nums[n-1]`，找到最大值。

这种方法的时间复杂度是 O(N*k)，其中 N 是数组的长度。当 N 和 k 都很大时，这个方法会超时。

#### 2. 优化的思路：单调队列

为了将时间复杂度降低到 O(N)，我们需要一种更高效的方法来获取窗口内的最大值，而不是每次都重新计算。这就是单调队列大显身手的地方。

**单调队列**是一种特殊的双端队列（Deque），它内部的元素（在这里我们存的是数组下标）所对应的值始终保持单调性（在这里是单调递减）。它的核心价值在于，队头元素永远是当前有效区间的最大值。

- **队列里存什么？**
  我们存放的是元素的**下标**，而不是元素的值。因为只存值，我们无法判断这个值是否已经滑出了窗口。通过下标，我们可以计算出它是否还在当前窗口的有效范围内。

- **如何维持单调性并确保结果正确？**
  当我们遍历到 `nums[i]` 时，窗口向右移动，我们需要对队列进行以下操作：

  1.  **移除过期队头**：
      检查队头元素的下标是否已经滑出当前窗口。当前窗口的范围是 `[i - k + 1, i]`。如果队头下标 `deque.peekFirst()` 小于等于 `i - k`，说明它已经不在窗口内，必须从队头 `pollFirst()` 移除。

  2.  **维护队尾单调递减**：
      从队尾开始，向前比较。如果队尾元素 `deque.peekLast()` 对应的 `nums` 值小于等于新元素 `nums[i]`，则将队尾元素 `pollLast()` 移除。重复此过程，直到队尾元素大于 `nums[i]` 或队列为空。
      *为什么这么做？* 因为如果一个队列中的元素 `nums[j]` (j < i) 比新来的 `nums[i]` 要小，那么只要 `nums[i]` 还在窗口内，`nums[j]` 就永远不可能成为窗口最大值了。因此，它就失去了成为"候选人"的资格，可以被淘汰。

  3.  **新元素入队**：
      经过上述清理，队列保证了单调性。此时将新元素的下标 `i` 从队尾 `offerLast()` 加入。

- **如何得到结果？**
  经过以上三步操作，**队头元素 `deque.peekFirst()` 对应的 `nums` 值，就始终是当前窗口的最大值**。当窗口已满（即 `i >= k-1`）时，我们就可以在每一步都记录下这个最大值。

#### 3. 算法步骤总结 (Canonical)

遍历数组 `nums` 从 `i = 0` 到 `n-1`：

1.  **移除过期队头**: 如果队列不为空，并且队头下标已经不在当前窗口 `[i-k+1, i]` 内 (即 `deque.peekFirst() <= i - k`), 则将其 `pollFirst()`。
2.  **维护队尾单调性**: 当队列不为空，并且队尾下标对应的 `nums` 值小于等于 `nums[i]` 时, 将队尾下标 `pollLast()`。
3.  **新元素下标入队**: 将当前下标 `i` `offerLast()` 到队列中。
4.  **记录结果**: 如果当前 `i` 已经可以构成一个完整的窗口 (即 `i >= k-1`), 那么队头 `deque.peekFirst()` 就是当前窗口的最大值，记录下来。

最终，这个算法每个下标最多入队一次、出队一次，所以时间复杂度是 O(N)，空间复杂度是 O(k)（队列最多存储 k 个元素）。

## 核心总结

- **问题本质**：在移动的区间内高效地求最值。
- **关键数据结构**：**单调队列**（通常用 `Deque` 实现）。
- **核心思想**：
    1.  用一个双端队列来维护一个单调递减（或递增）的序列。
    2.  队列中存储**下标**而不是值，以便判断元素是否已滑出窗口。
    3.  通过维护队列的单调性，使得队头元素永远是当前窗口的最值，从而实现 O(1) 的查询。
- **代码模板要点**：
    1.  `pollFirst`：移除过期的队头。
    2.  `while` + `pollLast`：从队尾移除破坏单调性的元素。
    3.  `offerLast`：加入新元素下标。
    4.  `peekFirst`：获取当前窗口最值。

## 举一反三

单调队列/单调栈是解决一类问题的通用模型，主要用于处理 "下一个更大/更小元素" 或 "区间最值" 的场景。

1.  **[739. 每日温度](https://leetcode.cn/problems/daily-temperatures/)**
    -   **相似点**：寻找每个元素右侧第一个比它大的元素。这可以看作是一个 "不固定窗口" 的查询。使用**单调栈**（栈顶到栈底单调递减），当新元素比栈顶大时，就为栈顶元素找到了答案。

2.  **[496. 下一个更大元素 I](https://leetcode.cn/problems/next-greater-element-i/)**
    -   **相似点**：和每日温度一样，是 "下一个更大元素" 问题的直接应用。可以用单调栈配合哈希表先预处理出 `nums2` 中所有元素的下一个更大元素，然后再查询 `nums1`。

3.  **[84. 柱状图中最大的矩形](https://leetcode.cn/problems/largest-rectangle-in-histogram/)**
    -   **相似点**：这道题是单调栈应用的经典难题。对于每个柱子，我们需要找到它左边和右边第一个比它矮的柱子，来确定以当前柱子为高的矩形的最大宽度。单调栈可以完美解决这个问题。

4.  **[1438. 绝对差不超过限制的最长连续子数组](https://leetcode.cn/problems/longest-continuous-subarray-with-absolute-diff-less-than-or-equal-to-limit/)**
    -   **相似点**：这道题是滑动窗口和单调队列的结合。它要求在窗口内 `max - min <= limit`。我们可以同时维护一个**单调递增队列**（用于快速获取窗口最小值）和一个**单调递减队列**（用于快速获取窗口最大值），然后移动窗口右指针，当不满足条件时收缩左指针。
