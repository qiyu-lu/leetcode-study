# 76. 最小覆盖子串

## 题目链接

[https://leetcode.cn/problems/minimum-window-substring/](https://leetcode.cn/problems/minimum-window-substring/)

## 思路解析

这道题要求我们在字符串 `s` 中，找出一个**最短的连续子串**，这个子串需要包含目标字符串 `t` 中的所有字符（包括数量）。

#### 1. 暴力解法

最直观的思路是找出 `s` 的所有子串，然后逐个判断它们是否满足条件，最后在所有满足条件的子串中找到最短的那个。

- 从 `s` 的第一个字符开始，遍历所有可能的子串。
- 对于每个子串，再检查它是否包含了 `t` 的所有字符。
- 记录下最短的有效子串。

这种方法需要 O(N^2) 的时间来生成所有子串，每次检查又需要额外的时间，整体效率非常低，在 `s` 长度很大时一定会超时。

#### 2. 优化的思路：滑动窗口

暴力解法的低效在于大量的重复计算。当我们检查完一个窗口 `s[i..j]` 后，再去检查 `s[i..j+1]` 时，其实大部分信息是可以复用的。这正是**滑动窗口**算法的用武之地。

我们可以用一对双指针 `left` 和 `right` 构成一个窗口，然后在 `s` 上滑动。

- **核心问题：如何高效判断窗口是否有效？**
  我们需要快速知道当前窗口内的字符是否已经覆盖了 `t` 的所有要求。这里可以使用两个哈希表（或数组）来解决：
    1.  `need` 表：用来统计 `t` 中所有字符的出现频率。
    2.  `window` 表：用来动态记录当前窗口 `[left, right)` 内各字符的出现频率。

- **如何进一步优化判断？**
  每次窗口变化都去完整比较 `window` 和 `need` 两个哈希表依然不够高效。我们可以引入一个 `valid` 变量来作为判断的“快捷方式”。`valid` 记录的是 `window` 中有多少**种类**的字符，其数量已经满足了 `need` 表的要求。当 `valid` 的值等于 `need` 表中不同字符的总数时，就说明这是一个有效的覆盖子串。

- **窗口如何滑动？**
  1.  **扩张阶段**：不断向右移动 `right` 指针，将新字符加入 `window`。如果这个新加入的字符使得 `window` 中某个字符的数量刚好满足了 `need` 的要求，我们就把 `valid` 加一。
  2.  **收缩阶段**：当 `valid` 值表明窗口已经有效时，我们就找到了一个可行解。但为了找到“最短”的，需要尝试收缩窗口。我们向右移动 `left` 指针，将左侧字符移出 `window`。如果被移出的字符导致 `window` 中某个字符的数量不再满足 `need` 的要求，我们就把 `valid` 减一，此时窗口失效。
  3.  **更新结果**：在每次收缩窗口**之前**（即窗口有效时），我们都计算一下当前窗口的长度，如果它比已记录的最小长度更短，就更新结果。

这个“扩张-收缩”的循环过程，保证了我们能不遗漏地找到全局最短的那个有效子串。

#### 3. 算法步骤总结

1.  初始化 `need` 哈希表，统计 `t` 的字符频率。初始化 `window` 表为空。
2.  初始化 `left = 0`, `right = 0`, `valid = 0`。以及用于记录结果的 `start` 和 `minLen`。
3.  `right` 指针开始向右遍历 `s`，进入扩张阶段。
4.  将 `s[right]` 字符计入 `window`。如果 `s[right]` 是 `t` 中需要的字符，并且它在 `window` 中的数量已满足 `need` 的要求，则 `valid++`。
5.  **当 `valid` 等于 `need` 中不同字符的总数时**，说明窗口有效，触发收缩阶段。
6.  在收缩前，先判断并更新最小子串的长度 `minLen` 和起始位置 `start`。
7.  将 `s[left]` 字符从 `window` 中移除，然后 `left++`。
8.  如果 `s[left]` 是 `t` 中需要的字符，并且它在 `window` 中的数量刚刚变得不满足 `need` 的要求，则 `valid--`，窗口失效，收缩暂停。
9.  重复步骤 3-8，直到 `right` 指针越过 `s` 的末尾。
10. 根据记录的 `start` 和 `minLen` 返回最终结果。

## 核心总结

- **问题本质**：在较长的字符串中，查找满足特定条件的**最短**连续子串。
- **关键数据结构**：**哈希表**（或数组），用于高效地进行字符频率统计（`need` 和 `window`）。
- **核心思想**：
    1.  **滑动窗口**：使用 `left` 和 `right` 双指针维护一个动态窗口。
    2.  **扩张与收缩**：`right` 指针负责扩张窗口以寻找可行解；`left` 指针在找到可行解后负责收缩窗口以寻找最优解。
    3.  **状态记录**：使用 `valid` 计数器来快速判断窗口的有效性，避免了在每一步都完整比较哈希表。
- **代码模板要点**：
    1.  `right++`：扩张窗口，更新 `window` 和 `valid`。
    2.  `while (valid == ...)`：进入收缩循环的条件。
    3.  更新结果（`minLen`, `start`）。
    4.  `left++`：收缩窗口，更新 `window` 和 `valid`。

## 举一反三

滑动窗口是解决子数组/子串问题的利器，核心思想不变，但“窗口有效性”的判断条件多种多样。

1.  **[3. 无重复字符的最长子串](https://leetcode.cn/problems/longest-substring-without-repeating-characters/)**
    -   **相似点**: 同样是滑动窗口，找最优子串。
    -   **不同点**: 窗口有效性变为“无重复字符”。当扩张窗口导致重复时，就需要收缩左边界。目标是找**最长**。

2.  **[438. 找到字符串中所有字母异位词](https://leetcode.cn/problems/find-all-anagrams-in-a-string/)**
    -   **相似点**: 字符频率的判断逻辑和本题几乎完全一样。
    -   **不同点**: 窗口大小是固定的，等于目标字符串的长度。因此，窗口更像是“滑动”而不是“伸缩”，`left` 和 `right` 指针同步右移。

3.  **[209. 长度最小的子数组](https://leetcode.cn/problems/minimum-size-subarray-sum/)**
    -   **相似点**: 和本题非常相似，都是找满足条件的**最短**连续子数组/子串。
    -   **不同点**: 判断条件从字符频率变成了“窗口内元素和大于等于目标值 `target`”。逻辑更简单，是练习滑动窗口思想的入门好题。

4.  **[1438. 绝对差不超过限制的最长连续子数组](https://leetcode.cn/problems/longest-continuous-subarray-with-absolute-diff-less-than-or-equal-to-limit/)**
    -   **相似点**: 滑动窗口找最优子数组。
    -   **不同点**: 窗口有效性是 `max - min <= limit`。为了在 O(1) 内获取窗口的最值，需要结合使用两个**单调队列**。这是滑动窗口与其它数据结构结合的复杂应用。
