# 算法笔记: _41_ 缺失的第一个正数

## 1. 解题思路

这道题的核心要求是以 O(n) 的时间复杂度和 O(1) 的常数级别额外空间来解决问题。O(1) 的空间复杂度提示我们不能使用哈希表等额外数据结构，而必须在**输入数组本身**上进行操作。

核心解法是**原地哈希**，也称为**原地交换**或**循环排序**。

思路如下：
我们希望将数组看作一个哈希表，理想的对应关系是：**数值 `k` 应该被放在索引为 `k-1` 的位置上**。例如，`1` 应该在 `nums[0]`，`2` 应该在 `nums[1]`，以此类推。

1.  **第一次遍历（整理数组）**:
    *   遍历数组，对于当前位置 `i` 的元素 `nums[i]`：
    *   我们不断地进行交换，直到 `nums[i]` 的值 `x` 被放到它正确的位置 `x-1` 上。
    *   这个交换的循环条件是 `1 <= nums[i] <= n` (确保要处理的数在有效范围内) 且 `nums[i] != nums[nums[i] - 1]` (确保它不在正确的位置，并且避免因重复元素导致死循环)。
    *   例如，如果 `nums[i] = 3`，我们就应该把它和 `nums[2]` 的元素交换。交换后，新的 `nums[i]` 可能仍然不在正确的位置，所以我们继续这个过程，直到 `nums[i]` 无需交换为止。

2.  **第二次遍历（查找结果）**:
    *   经过第一次遍历后，数组已经部分有序。理想情况下，`nums[i]` 的值应该是 `i+1`。
    *   我们再次从头遍历数组，找到第一个不满足 `nums[i] == i + 1` 的索引 `i`。那么 `i+1` 就是我们寻找的缺失的第一个正数。
    *   如果整个数组都满足 `nums[i] == i + 1`，说明从 `1`到 `n` 都存在，那么缺失的第一个正数就是 `n+1`。

*   **时间复杂度**: O(n)。虽然看起来是嵌套循环，但每个元素最多被交换到其正确位置一次。所以总的交换次数不会超过 n 次，因此是线性时间。
*   **空间复杂度**: O(1)。我们只在原数组上进行操作，没有使用任何额外的存储空间。

## 2. 思路来源 (如何想到)

1.  **空间复杂度约束**: O(1) 的额外空间是本题最关键的约束。它直接排除了使用哈希表、新数组等需要 O(n) 空间的方法。这迫使我们思考如何在原数组上做文章。
2.  **数值范围的暗示**: 题目要找的是“缺失的第一个**正数**”。对于一个长度为 `n` 的数组，它能容纳的正数最多是 `1, 2, ..., n`。因此，我们要找的答案必定在 `[1, n+1]` 这个范围内。
3.  **下标与数值的联系**: 既然答案在 `[1, n+1]` 范围内，而数组的下标范围是 `[0, n-1]`，这自然地启发我们将**数值 `k`** 和**下标 `k-1`** 关联起来。这个关联是构建“原地哈希”的基础。
4.  **最终推导**: 结合以上几点，我们可以得出：利用数组下标作为“桶”，将数值作为“元素”放入对应的桶中。遍历一遍数组，将所有在 `[1, n]` 范围内的元素归位。然后再遍历一遍，第一个“桶”里没有对应“元素”的，就是缺失的数。

## 3. 举一反三

这种“原地哈希”或利用下标与数值对应关系的思想在数组类问题中很常见。

1.  **_287_ Find the Duplicate Number (寻找重复数)**:
    *   **联系**: 同样是利用数值和下标的映射关系。虽然这道题通常用快慢指针（弗洛이드判圈算法）来解决，但也可以用原地交换的思路。尝试将数值 `k` 放到下标 `k` (或 `k-1`)，如果在交换前发现目标位置上已经有了正确的数，那么当前数就是重复数。

2.  **_448_ Find All Numbers Disappeared in an Array (找到所有数组中消失的数字)**:
    *   **联系**: 思路几乎一样。遍历数组，将遇到的数值 `x` 对应的位置 `x-1` 上的数标记为负数。第二次遍历时，所有值仍为正数的位置 `i`，其对应的 `i+1` 就是消失的数字。这是原地哈希的另一种实现方式（通过改变符号来标记）。

3.  **_268_ Missing Number (丢失的数字)**:
    *   **联系**: 题目范围是 `[0, n]`，同样是找缺失的数。可以用原地交换（将 `k` 放到 `nums[k]`），也可以用位运算（异或）或高斯求和公式来更巧妙地解决。但其核心问题模型是相似的。

## 4. 涉及的关键方法/API

这个算法的核心在于手动实现原地交换逻辑，因此不依赖特定的库函数。唯一涉及的操作就是基本的数组读写和变量交换。

*   **数组访问**: `nums[i]`
*   **变量交换**:
    ```java
    int temp = nums[a];
    nums[a] = nums[b];
    nums[b] = temp;
    ```
