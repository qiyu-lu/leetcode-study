# 算法笔记: 189. 轮转数组 (Rotate Array)

作为一名资深算法工程师，我将为你深入总结这道题。

## 1. 解题思路

这道题最经典且空间最优的解法是 **数组翻转**，这是一种非常巧妙的原地算法。

核心思路分为三步：
1.  **整体翻转**：将整个数组的所有元素进行翻转。
2.  **局部翻转 (前 k 个)**：将数组的前 `k` 个元素（`k` 是对数组长度取模后的值）进行翻转。
3.  **局部翻转 (后 n-k 个)**：将数组的剩余 `n-k` 个元素进行翻转。

**举例说明**: `nums = [1,2,3,4,5,6,7], k = 3`
1.  `k` 对长度 7 取模后仍为 3。
2.  **整体翻转**后: `[7,6,5,4,3,2,1]`
3.  **翻转前 k=3 个元素**后: `[5,6,7,4,3,2,1]`
4.  **翻转后 n-k=4 个元素**后: `[5,6,7,1,2,3,4]` -> 得到最终结果。

*   **时间复杂度**: O(n)。数组中的每个元素都被访问和交换了常数次（具体是两次翻转，一次在整体中，一次在局部中），因此总时间与数组长度成线性关系。
*   **空间复杂度**: O(1)。该算法是原地操作，仅使用了几个临时变量进行交换，不需要额外的存储空间。

## 2. 思路来源 (如何想到)

题目的目标是将数组的末尾部分移动到开头，并且进阶要求使用 O(1) 的额外空间，这提示我们不能使用辅助数组，必须进行**原地操作**。

1.  **观察结构**: 旋转后的数组 `[5,6,7,1,2,3,4]` 实际上是原数组的两段 `[1,2,3,4]` (记为 A) 和 `[5,6,7]` (记为 B) 交换了位置。即原始形态是 `[A, B]`，目标形态是 `[B, A]`。
2.  **联想翻转**: "翻转" 是一个强大的原地操作工具，我们自然会思考如何用翻转实现 `[A, B]` -> `[B, A]` 的变换。
3.  **推导技巧**: 这里有一个经典的变换技巧：`(X^r Y^r)^r = Y X`，其中 `^r` 代表翻转操作。
    *   如果我们对整个数组 `[A, B]` 翻转，会得到 `[B^r, A^r]`。
    *   此时，数组的前半部分是 `B` 的翻转，后半部分是 `A` 的翻转。
    *   再分别对 `B^r` 和 `A^r` 进行一次翻转，就能将它们恢复成 `B` 和 `A`，最终得到 `[B, A]`。
    *   这个推导过程恰好对应了上述的三步翻转法，逻辑清晰且高效。

## 3. 举一反三

这种 "三步翻转法" 的技巧在字符串和数组操作中非常有用，特别是涉及原地旋转和块交换的场景。

1.  **LeetCode 151. 反转字符串中的单词 (Reverse Words in a String)**:
    *   **联系**: 思路几乎完全一样。先将整个字符串翻转，然后遍历字符串，再将每个单词（由空格分隔）单独翻转回来。
2.  **剑指 Offer 58 - II. 左旋转字符串**:
    *   **联系**: 这是本题的直接变体，要求向左旋转。解法完全相同，只是分割点 `k` 的定义不同。向左旋转 `k` 位等价于向右旋转 `n-k` 位。
3.  **LeetCode 48. 旋转图像 (Rotate Image)**:
    *   **联系**: 虽然是二维数组，但其核心思想也是通过翻转（例如，先水平翻转，再沿对角线翻转）来实现原地旋转90度，体现了通过翻转变换实现复杂原地操作的共同思想。

## 5. 关于我的解法

你提供的解法使用了辅助数组 `temp`，这是一种非常直观且正确的思路。它首先计算出每个元素旋转后的最终位置，然后构建一个新的 `temp` 数组，最后再将 `temp` 数组的内容复制回原数组。

这种方法的时间复杂度是 O(n)，空间复杂度是 O(n)。在面试中，这通常是一个可以接受的初步答案。面试官接下来很可能会追问：“能否用 O(1) 的额外空间来解决这个问题？”，这时就自然引出了我们上面讨论的 "数组翻转" 方法。你的解法没有逻辑问题，只是在空间效率上可以进一步优化。
