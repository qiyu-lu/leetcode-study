# 算法笔记: 238. 除自身以外数组的乘积 (Product of Array Except Self)

你好，作为一名资深算法工程师，我非常理解你首先想到除法思路，这是最直接的数学逻辑。但这道题的精髓恰恰在于 **“不能使用除法”** 这一约束，它迫使我们寻找一种更具创造性的解法。

## 1. 解题思路

核心思路是 **前缀积与后缀积的结合**。对于数组中的任何一个位置 `i`，它最终的结果 `answer[i]` 等于它 **左边所有元素的乘积** 乘以它 **右边所有元素的乘积**。

### 解法一：使用辅助数组 (O(n) 空间)

1.  创建两个数组，`prefix` 和 `suffix`，并初始化所有值为 1。
2.  `prefix[i]` 存储 `nums[0]...nums[i-1]` 的乘积。我们从左到右遍历 `nums` 来填充 `prefix` 数组。
3.  `suffix[i]` 存储 `nums[i+1]...nums[n-1]` 的乘积。我们从右到左遍历 `nums` 来填充 `suffix` 数组。
4.  最后，`answer[i] = prefix[i] * suffix[i]`。

*   **时间复杂度**: O(n)。需要三次独立的线性遍历（计算前缀积、计算后缀积、计算最终结果）。
*   **空间复杂度**: O(n)。需要额外的 `prefix` 和 `suffix` 数组。

### 解法二：空间优化 (O(1) 空间)

这是本题的进阶要求，我们可以利用输出数组 `answer` 来优化空间。

1.  **第一遍遍历 (从左到右)**:
    *   初始化一个变量 `prefix_prod = 1`。
    *   遍历数组，对于位置 `i`，先将 `prefix_prod` 的值赋给 `answer[i]`，然后将 `prefix_prod` 乘以 `nums[i]`。
    *   这遍结束后，`answer[i]` 存储的是 `i` 左侧所有元素的乘积。

2.  **第二遍遍历 (从右到左)**:
    *   初始化一个变量 `suffix_prod = 1`。
    *   从右向左遍历数组，对于位置 `i`，先将 `answer[i]` 乘以 `suffix_prod`，然后将 `suffix_prod` 乘以 `nums[i]`。
    *   这一步相当于在之前计算的左侧乘积的基础上，又乘上了右侧的乘积，得到最终结果。

*   **时间复杂度**: O(n)。需要两次线性遍历。
*   **空间复杂度**: O(1)。题目说明返回的 `answer` 数组不计入额外空间复杂度，我们只用了两个额外变量。

## 2. 思路来源 (如何想到)

1.  **核心约束 -> 另辟蹊径**: "不能使用除法" 是最强的信号，它告诉我们必须通过 **构造** 的方式来得到结果，而不是通过 "排除" (除法) 的方式。
2.  **分解问题**: `answer[i]` 的值由两部分构成：`i` 左边的部分和 `i` 右边的部分。这是一个非常自然的分解：
    `answer[i] = (nums[0] * ... * nums[i-1]) * (nums[i+1] * ... * nums[n-1])`
3.  **识别重复计算**: 如果为每个 `i` 都独立计算左右两边的乘积，会产生大量的重复计算，复杂度为 O(n²)。优化的关键在于识别到 `prefix[i]` 和 `prefix[i-1]` 之间，以及 `suffix[i]` 和 `suffix[i+1]` 之间的递推关系。
4.  **前缀和/积模式**: 这种 "依赖于左/右侧聚合值" 的问题，是 **前缀和 (Prefix Sum)** 模式的典型变体。一旦识别出这个模式，就可以联想到通过预计算前缀和/积、后缀和/积来将 O(n²) 复杂度降至 O(n)。

## 3. 举一反三

这种 "左右开弓"、"两遍扫描" 的思想在数组问题中非常常见。

1.  **LeetCode 42. 接雨水 (Trapping Rain Water)**:
    *   **联系**: 经典题目。在位置 `i` 能接多少雨水，取决于它左边的最高墙 `max_left` 和右边的最高墙 `max_right`。计算 `answer[i]` 依赖于左右两边的信息，和本题的模式如出一辙。也可以通过 O(n) 空间（左右最大值数组）或 O(1) 空间（双指针）解决。
2.  **LeetCode 135. 分发糖果 (Candy)**:
    *   **联系**: 一个孩子需要分的糖果数，既要比左边评分低的孩子多，也要比右边评分低的孩子多。这需要两遍扫描来解决：第一遍从左到右，保证比左边多；第二遍从右到左，保证比右边多。
3.  **LeetCode 53. 最大子数组和 (Maximum Subarray)**:
    *   **联系**: 虽然解法不同（动态规划），但它也体现了 "前缀和" 思想的演化。通过维护一个局部的最优解，并不断向右扩展，最终找到全局最优。

## 4. 涉及的关键方法/API

本题的算法思想远比 API 使用重要。

*   **核心操作**: 主要是基本的数组读写 `nums[i]` 和乘法 `*`。
*   **数组初始化**: 在使用辅助数组时，可以用 `Arrays.fill(prefix, 1)` 来进行初始化，这比循环更简洁。
