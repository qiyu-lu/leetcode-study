# 56. 合并区间 (Merge Intervals)

## 核心思想

这道题是区间问题的经典题目。核心思想是**贪心算法**，通过排序来简化问题。

1.  **排序**：首先，将所有区间按照**左端点**从小到大进行排序。这一步是关键，它保证了我们处理区间时，是按照顺序向前推进的，无需回头考虑之前的区间。
2.  **合并**：排序后，我们维护一个已经合并好的结果列表 `merged`。遍历排序后的所有区间：
    *   如果 `merged` 为空，或者当前遍历到的区间 `current` 与 `merged` 中的最后一个区间 `last` **不重叠**，那么 `current` 就是一个新的、独立的区间，直接将其加入 `merged`。
    *   如果 `current` 与 `last` **重叠**，说明它们可以合并。我们更新 `last` 的右端点，使其等于 `max(last.end, current.end)`。因为我们已经按左端点排序，所以合并后的新区间左端点一定是 `last.start`。

判断两个区间 `[start1, end1]` 和 `[start2, end2]` (已按 start 排序，`start1 <= start2`) 是否重叠的条件非常简单：`start2 <= end1`。

**示例**: `[[1,3],[2,6],[8,10],[15,18]]`

1.  **排序**: 数组已经按左端点排好序了。
2.  **合并**:
    *   初始化 `merged = [[1,3]]`。
    *   处理 `[2,6]`：`2 <= 3` (重叠)，更新最后一个区间为 `[1, max(3, 6)]` -> `[1, 6]`。现在 `merged = [[1,6]]`。
    *   处理 `[8,10]`：`8 > 6` (不重叠)，直接添加。现在 `merged = [[1,6], [8,10]]`。
    *   处理 `[15,18]`：`15 > 10` (不重叠)，直接添加。现在 `merged = [[1,6], [8,10], [15,18]]`。
3.  **完成**: 最终结果为 `[[1,6], [8,10], [15,18]]`。

## 举一反三：相似问题

掌握了“排序+处理重叠区间”的思想，可以解决一系列相关问题：

1.  **[LeetCode 57. 插入区间](https://leetcode.cn/problems/insert-interval/)**:
    *   **问题**: 在一个不重叠的、已排序的区间列表中插入一个新区间，并合并。
    *   **联系**: 这是合并区间的直接变体。可以先将新区间找到合适的位置插入，然后就变成了标准的“合并区间”问题。更优的做法是一次遍历，找到新区间影响的范围，然后合并。

2.  **[LeetCode 435. 无重叠区间](https://leetcode.cn/problems/non-overlapping-intervals/)**:
    *   **问题**: 计算需要移除的最少区间数，使得剩余区间不重叠。
    *   **联系**: 贪心思想。但这次通常是按**右端点**排序。策略是“尽量早地结束当前区间，为后面的区间留出更多空间”，所以每次都保留右端点最小的那个。

3.  **[LeetCode 253. 会议室 II](https://leetcode.cn/problems/meeting-rooms-ii/)**:
    *   **问题**: 给定一系列会议的开始和结束时间，计算最少需要多少个会议室。
    *   **联系**: 问题可以转化为“在同一时刻，最多有多少个重叠的会议”。可以通过将所有区间的开始时间和结束时间放在一起排序，遇到开始时间就+1（需要一个会议室），遇到结束时间就-1（释放一个会议室），这个过程中的最大值就是答案。通常用最小堆（Min-Heap）来做会更直观，堆里存放会议的结束时间。

4.  **[LeetCode 986. 区间列表的交集](https://leetcode.cn/problems/interval-list-intersections/)**:
    *   **问题**: 给定两个已排序且不重叠的区间列表，求它们的交集。
    *   **联系**: 使用双指针法。两个指针分别指向两个列表的当前区间，比较并计算交集，然后移动指针（哪个区间的右端点小就移动哪个指针）。

5.  **[LeetCode 763. 划分字母区间](https://leetcode.cn/problems/partition-labels/)**:
    *   **问题**: 将一个字符串划分为尽可能多的片段，使得每个字母只出现在一个片段中。
    *   **联系**: 这个问题可以转化为区间问题。首先遍历一遍字符串，记录每个字母最后出现的位置。这样，对于字符串中的每个字符，我们就有了一个隐式的“区间”`[start, end]`，`start`是它第一次出现的位置，`end`是它最后一次出现的位置。然后问题就变成了合并这些字符的区间。

## Java 知识点补充

在你的解法中，有几个值得注意的 Java 语法和 API 调用，这里展开说明一下。

### 1. 使用 Lambda 表达式进行排序

代码: `Arrays.sort(intervals, (a, b) -> a[0] - b[0]);`

*   `Arrays.sort()` 方法可以接收一个额外的 `Comparator` (比较器) 参数，用来定义自定义的排序规则。
*   ` (a, b) -> a[0] - b[0]` 是一个 **Lambda 表达式**，它是 `Comparator` 接口一个简洁的实现方式。
    *   `a` 和 `b` 代表数组中要比较的两个元素，在这里它们都是 `int[]` 类型的数组 (例如 `[1,3]` 和 `[2,6]`)。
    *   `->` 后面是比较逻辑。`a[0] - b[0]` 表示用第一个元素的第一个数字减去第二个元素的第一个数字。
    *   **排序规则**:
        *   如果结果为**负数**，表示 `a`应该排在`b`的前面。
        *   如果结果为**正数**，表示 `b`应该排在`a`的前面。
        *   如果结果为**零**，`a`和`b`的相对位置不变。
*   所以，这行代码的含义是：按照每个区间 (子数组) 的**第一个元素 (左端点) `[0]`** 进行**升序**排序。

### 2. List 转换为二维数组

代码: `return ans.toArray(new int[ans.size()][]);`

*   在算法题中，我们经常使用 `ArrayList` 因为它的长度是动态可变的，方便添加元素。但题目最终要求返回一个 `int[][]` 类型的数组。
*   `ans.toArray()` 方法就是用来实现这个转换的。
*   它接收一个数组作为参数，这个参数的作用是**指定返回数组的类型**。
    *   `new int[ans.size()][]` 创建了一个和列表大小 `ans.size()` 相同的新二维数组 `int[][]`。
    *   `toArray` 方法会把 `List` 中的所有元素（这里是 `int[]`）填充到这个新创建的数组中，并将其返回。
*   如果你传入的数组容量足够大，`toArray`会直接使用你传入的数组；如果不够大（比如 `new int[0][]`），它会根据列表的大小和传入数组的类型，反射创建一个新的、大小合适的新数组。因此 `ans.toArray(new int[0][])` 也是一种常见的写法。
