# 53. 最大子数组和 (Maximum Subarray)

### 题目链接

[LeetCode 53](https://leetcode.cn/problems/maximum-subarray/)

### 核心思想

这道题是动态规划的经典入门题目。核心思想是**局部最优解**和**全局最优解**的更新。

当我们遍历数组时，对于当前元素 `nums[i]`，我们需要考虑以它为结尾的连续子数组的最大和是多少。这个最大和要么是 `nums[i]` 本身（即开启一个新的子数组），要么是 `nums[i]` 加上前一个元素为结尾的连续子数组的最大和（即扩展当前的子数组）。

通过维护一个`局部最优解`（以当前元素结尾的最大和）和一个`全局最优解`（到目前为止遇到的最大和），我们就可以在一次遍历中找到答案。

### 解法

#### 方法一：动态规划 (Kadane's Algorithm)

这是最经典、最高效的解法。

1.  **定义状态**:
    `dp[i]` 表示以 `nums[i]` **结尾**的连续子数组的最大和。

2.  **状态转移方程**:
    在计算 `dp[i]` 时，我们有两个选择：
    *   将 `nums[i]` 加入到前面的子数组中，得到 `dp[i-1] + nums[i]`。
    *   从 `nums[i]` 开始一个新的子数组，值为 `nums[i]`。
    因此，状态转移方程为：
    `dp[i] = max(nums[i], dp[i-1] + nums[i])`

3.  **初始化**:
    `dp[0] = nums[0]`

4.  **结果**:
    最终的答案不是 `dp[n-1]`，而是整个 `dp` 数组中的最大值，因为最大和的子数组可能在任何位置结束。
    `result = max(dp[0], dp[1], ..., dp[n-1])`

**空间优化**:
我们可以注意到 `dp[i]` 的计算只依赖于 `dp[i-1]`。因此，我们不需要一个完整的 `dp` 数组，只需要一个变量来记录前一个状态即可。

```java
public int maxSubArray(int[] nums) {
    int maxSoFar = nums[0]; // 全局最优解
    int maxEndingHere = nums[0]; // 局部最优解

    for (int i = 1; i < nums.length; i++) {
        // 以当前元素结尾的最大和
        maxEndingHere = Math.max(nums[i], maxEndingHere + nums[i]);
        // 更新全局最大和
        maxSoFar = Math.max(maxSoFar, maxEndingHere);
    }
    return maxSoFar;
}
```

#### 方法二：前缀和

这个方法也很巧妙，利用了子数组和可以用两个前缀和的差来表示的性质。

`sum(i, j) = prefixSum[j] - prefixSum[i-1]`

当我们遍历到 `j` 时，我们想找到一个 `i <= j`，使得 `prefixSum[j] - prefixSum[i-1]` 最大。为了实现这一点，我们只需要在 `0` 到 `j-1` 的范围内找到一个最小的 `prefixSum` 值。

1.  计算前缀和数组 `pre`。
2.  遍历数组，对于每个位置 `i`，计算当前的前缀和 `pre[i+1]`。
3.  在计算 `pre[i+1]` 的同时，维护一个 `preMin`，记录 `pre[0]` 到 `pre[i]` 的最小值。
4.  那么在 `i` 位置，以 `nums[i]` 结尾的某个子数组能取得的最大和就是 `pre[i+1] - preMin`。
5.  用一个全局变量 `ans` 更新所有 `pre[i+1] - preMin` 中的最大值。

这种方法与你代码中的思路一致，也是 O(n) 的时间复杂度和 O(n) 的空间复杂度（如果创建前缀和数组）或 O(1) 的空间复杂度（如果不创建，只维护一个变量）。

### 举一反三：相似题目

这类 "最大/最小连续子数组..." 问题是动态规划和数组问题的常见模式。

1.  **[LeetCode 152. 乘积最大子数组](https://leetcode.cn/problems/maximum-product-subarray/)**
    *   **相似之处**: 思路与最大子数组和非常相似。
    *   **不同之处**: 因为负数的存在（负负得正），我们在维护“以当前元素结尾的最大积”的同时，还需要维护一个“以当前元素结尾的最小积”。状态转移会变得更复杂，需要同时考虑最大值和最小值。

2.  **[LeetCode 918. 环形子数组的最大和](https://leetcode.cn/problems/circular-subarray-maximum-sum/)**
    *   **相似之处**: 直接应用 Kadane's 算法。
    *   **不同之处**: 数组是环形的。最大和的子数组有两种情况：
        1.  不跨越数组边界，就是普通的最大子数组和（用 Kadane's 算法）。
        2.  跨越数组边界（例如，一部分在末尾，一部分在开头）。这种情况等价于 `数组总和 - 最小子数组和`。最小子数组和可以通过将数组所有元素取反，然后求最大子数组和得到。
        最后比较这两种情况取最大值。

3.  **[LeetCode 1191. K 次串联后最大子数组之和](https://leetcode.cn/problems/k-concatenation-maximum-subarray-sum/)**
    *   **相似之处**: 核心还是求最大子数组和。
    *   **不同之处**: 数组被重复了 k 次。需要分情况讨论：
        *   `k=1`：就是原问题。
        *   `k>1`：最大和可能来自：1) 单个数组内部的最大子数组和；2) 跨越两个数组边界（`array_suffix_max + array_prefix_max`）。如果整个数组的和 `totalSum > 0`，那么还可以把中间 `k-2` 个数组全选上。

这些题目都围绕着 Kadane's 算法的核心思想进行扩展和变形，考察对问题不同情况的分析能力。
